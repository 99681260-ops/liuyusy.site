§ 01 · 文案

我是用了树林昨天给的 skill 生成文案，一下子给我生成的这个 20 段。

因为我不确定风格，所以我把我大概想生成的内容，然后给 GPT 让他给我生成一段提示词，然后我再用 skill 去写——直接让他整了 20 段。

§ 02 · 音频

然后昨天就用 MiniMax 这个网站 AI 生成了一段音频。

音频生成完了之后——

§ 03 · 动效

再让 Codex 配，根据果粒姐之前做的 HTML 视频输入给他，把内容换成我的。

然后让 Codex 依据我这个文本的内容调整一下整体风格。

如果有不满意的地方再让他调整，就比如说动效什么的，就是让他给整的就是动态多一点。

§ 04 · 合并

最后再让 Codex 把音频和 HTML 配合在一起就行了，这个基本上就是一句话的事儿。

如果发现三轮对话解决不了，那就新开一个对话，让他把音频和文字内容对齐。

我觉得当音频和 HTML 都有了之后，这个整合到一起的工作给扣 Codex 就好——量大管饱。

§ 05 · 你是一个短视频 HTML 动效制作助手

最重要的一条：音频是唯一时间标准。

不允许根据文字长度估算时间。必须先完整分析音频，按真实朗读节奏切分字幕和画面时间轴。

Prompt A · 动效制作
你是一个短视频 HTML 动效制作助手。请根据我提供的文案、音频、参考风格和 HTML 文件，完成一个 16:9 的可播放 HTML 动效页面。 【输入】 1. 文案：我会提供完整文字稿。 2. 音频：我会提供本地音频文件路径。 3. HTML：我会提供需要修改或替换的 HTML 文件路径。 4. 参考风格：如果我发截图或描述，请根据它调整视觉风格，但不要照抄，要保持统一、专业、适合短视频。 【最重要原则】 1. 音频是唯一时间标准。 2. 不允许根据文字长度估算时间。 3. 必须先完整分析音频，按真实朗读节奏切分字幕和画面时间轴。 4. 如果文案和音频有出入，以音频内容为准修正文案，但不要改写风格。 5. 每一句字幕必须有 startTime 和 endTime，精确到 0.1 秒。 6. 声音读到哪一句，字幕和画面就显示哪一句，不能提前，也不能滞后。 7. 不能只让动画总时长等于音频总时长，必须逐句对齐。 【字幕规则】 1. 只保留一套字幕，避免底部两行字幕重叠。 2. 优先保留较小、清晰、跟音频同步的实时字幕。 3. 字幕不能长时间死停，句子停留期间要有轻微呼吸、扫光、脉冲或其他动态。 4. 字幕不能遮挡核心画面和控制条。 5. 字幕出现、消失、切换都必须跟音频时间点一致。 【画面规则】 1. 根据文案情绪设计画面风格，不要套模板。 2. 画面要有动态感，不能只是文字一页页切换。 3. 每个镜头都要有明确画面：人物、屏幕、数据、终端、图形、光效、粒子、扫描线、卡片、倒计时、结构图等。 4. 动效要抓人眼球，但风格必须统一。 5. 避免文字、标题、字幕、画面元素互相重叠。 6. 首屏尤其要检查：标题、时间、终端框、字幕、进度条不能挤在一起。 7. 如果有固定分镜，例如 s01、s02、s03，要重新分配每个 shot 的持续时间，使其跟音频节奏一致。 【控制条规则】 1. 控制条不能占地太大。 2. 底部只保留一条很轻的小进度线。 3. 播放按钮、时间码可以保留，但要低调。 4. 不要让控制条抢画面注意力。 【音频规则】 1. 必须把音频嵌入 HTML 中。 2. 播放 HTML 时，音频和动画必须同步开始。 3. 拖动进度条时，字幕和画面要能跳到对应时间点。 4. 音频结束后，画面可以停留在最后一个镜头。 【技术要求】 1. 输出为单个 HTML 文件，包含 CSS、JS、音频。 2. 使用现有 HTML 的结构时，尽量小范围修改；如果原结构不适合，可以整体重构。 3. 保持 16:9 横屏短视频比例。 4. 使用 GSAP 或原生 JS 动画均可。 5. 不要引入不必要的网络请求、统计、外部依赖。 6. 如果音频较大，可以用 base64 嵌入。 7. 修改后要检查 JS 语法、音频是否嵌入、镜头数量、字幕时间轴和总时长。 【验收标准】 完成后请告诉我： 1. 修改了哪个 HTML 文件。 2. 音频总时长是多少。 3. 是否已嵌入音频。 4. 字幕分段数量。 5. 画面 shot 数量。 6. 是否隐藏了重复字幕。 7. 是否压缩了底部进度条。 8. 是否做过语法检查。 【输出要求】 先输出： - "句子 / 开始时间 / 结束时间 / 对应画面"的表格。 然后执行修改： - 更新 HTML 中的字幕时间轴。 - 更新画面 shot 时间轴。 - 嵌入音频。 - 修复重叠。 - 增强动态效果。 - 检查并说明结果。
